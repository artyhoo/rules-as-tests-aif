## §4 Per-class audit findings

### OWN-BUILD (full depth: external-lit sweep + problem-class + alternative-enumeration)

**The single load-bearing OWN-BUILD cluster = the 30 principle tests** (`packages/core/principles/*.test.ts`). These are the literal embodiment of the project thesis: executable tests that fail when a discipline-bearing markdown doc (rule, skill, agent, kickoff, research-patch) violates a structural manifest. Audited to OWN-BUILD depth:
- **Problem class:** "assert that the project's own prose-convention documents carry required structure (authority header, paired-negative block, scope annotation, prior-art citation, T-enumeration) — deterministically, no LLM, fail at CI."
- **External alternative sweep:** §6 NE-1. Result: NO drop-in. ArchUnit/NetArchTest = code-architecture only (DeepWiki confirmed ArchUnit cannot process markdown). conftest/OPA = structured-config only, would need a custom markdown→JSON extractor (= the BUILD slice we own). markdownlint = lint syntax, not semantic manifest-conformance. Superpowers TDD-for-skills = tests *agent behavior in headless mode* (LLM-in-CI, violates [no-paid-llm-in-ci.md](../../../.claude/rules/no-paid-llm-in-ci.md)), not static structural conformance.
- **Alternative if stripped:** revert to prose-only discipline + reviewer vigilance — exactly the pre-project state the README thesis exists to replace. No mechanical substitute. **Verdict: OWN-BUILD correctly held; escalation-to-OWN-BUILD-depth confirms novelty of problem-class, not just absence-in-usual-sources.**

**14 hooks (CC-native bash):** the hooks API itself is ADOPT (SSOT #20). The 14 scripts are own-build edit-time/session-time enforcement (earliest-channel delivery). Alternative if stripped: the same checks shift right to pre-push/CI (later channel) — degraded but not lost, since most hooks have a pre-push or principle-test backstop. Two unregistered hooks (`adopt-orchestrator-prompts.sh`, `worktree-setup.sh`) fire on non-settings.json channels (manual / `claude -w`) — **not dead, but invisible to a settings.json-only audit** (finding F1, §8).

**pre-push own-build arms (priorArt, s17, unpinnedToolInstall):** deterministic regex/trailer parsers. Alternative if stripped: CI-only enforcement (discipline-self-check.yml duplicates s17). Backstop exists.

### ADAPTED (sweep modifications + T16 problem-class match)

**Custom AST eslint-rules (R2/R7/R12/R8/R14/R20):** the engine-assignment-matrix (#698 R-phase, 2026-06-23) already ran the full 2-engine sweep (esquery vs ast-grep) on all of these and recorded per-rule verdicts with verbatim ESLint output. T16 match: **upstream problem class = "lint AST patterns"; our problem class = "framework-aware AI-generated-code convention enforcement with `audit:exempt` + path-scoping". Match: PARTIAL** — the engine (esquery) is reused verbatim, the rule *logic* (exempt-comment handling, glob-scoping) is the ADAPTED delta the engines "can't express" (matrix verdict KEEP-CUSTOM for R2/R7/R12). Evidence: matrix §2 R2 note line 122, R7 line 182, R12 line 196.

**guard-liveness pre-push arms:** ADAPT of SSOT #114 change-scoped ESLint negative-test roundtrip. T16 match: YES — upstream pattern (RuleTester negative roundtrip) == our "prove the rule still fires on a known-bad fixture" problem.

### ADOPTED-MECHANISM (5-check integration audit) — sample

R-rules delegating to 3rd-party (R5 no-floating-promises, R9 no-restricted-imports, R15 jsx-a11y): (1) problem-class match YES — they ARE the lint problem; (2) versions A5-scope; (3) upstream maintained (typescript-eslint, jsx-a11y active 2026) A5-scope; (4) integration = standard ESLint config, best-practice; (5) drop-in = swap rule, trivial. **No T13 escalation needed** — these are mainstream, externally-validated, NOT context7-only.

### ADOPTED-VOCABULARY (sanity only)

Trailer-syntax (commitlint/git interpret-trailers vocab, SSOT #23/#26 ADOPT-VOCABULARY): we use `Prior-art:`/`Co-authored-by:` RFC822 trailer naming; implementation is our own parser (`prior-art.ts`). No false-precedent — SSOT already records "ADOPT VOCABULARY, impl ours". Consistent.
