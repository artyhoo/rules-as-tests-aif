## §2 Classification rubric

Anchored on the project's own tautology definition ([04-no-tautology.test.ts:51-86](../../../packages/core/principles/04-no-tautology.test.ts) — no-op command / policy==title / bad==good) and the «documents lie; tests don't» thesis (a principle test must FAIL when its discipline is violated):

- **REAL-ASSERTION** — reads a real artifact (manifest / repo files / config / git) AND asserts a non-trivial property that would FAIL on a real violation; strength-bonus for a paired-negative / `mutation:` arm feeding deliberately-broken input to the same checker.
- **SUSPECT-TAUTOLOGY** — expected derived from the same source as actual; OR always-true on a constructed value; OR the looped population can be EMPTY with no non-vacuity sentinel (`0 items → 0 violations → green`); OR the only «negative» arm mutates a local literal and never runs the real checker against the real tree; OR pure self-reference (asserts the test file's own text).
- **NO-OP** — asserts nothing meaningful (`.toBeDefined()` on always-defined, `expect(true).toBe(true)`, skipped/empty, asserts a freshly-built value equals itself).

## §4 Parked findings (DO-NOT-REWRITE — surfaced for maintainer disposition)

Per the Task-3 mandate and [reviewer-discipline.md §1](../../../.claude/rules/reviewer-discipline.md), these are surfaced, not fixed — rewriting a discipline principle-test is a separate judgement call.

### §4.1 DN-T3-1 — `16-hook-stub-completeness.test.ts` is vacuous against the current tree (SUSPECT)

- **Evidence:** `runStubCompletenessCheck` short-circuits at [16:102](../../../packages/core/principles/16-hook-stub-completeness.test.ts) — `if (hardFailScripts.length === 0) return [];` — and `pre-push.ts` currently has **0** `packages/core/audit-self/*.test.sh` hard-fail references (`grep -coE 'packages/core/audit-self/[a-z0-9-]+\.test\.sh' packages/core/hooks/pre-push.ts` → `0`, verified 2026-06-27). The file's own docstring [16:18-22](../../../packages/core/principles/16-hook-stub-completeness.test.ts) concedes it: «an empty hard-fail set is VACUOUSLY VALID — the test PASSES».
- **What slips past today:** the real-tree arm ([16:185-186](../../../packages/core/principles/16-hook-stub-completeness.test.ts)) asserts nothing about the current tree; the stub-matching loop is exercised only against synthetic `FIXTURE_*` literals. The invariant only re-arms if a `.test.sh` hard-fail re-enters `pre-push.ts`.
- **Option A** → add a non-vacuity sentinel that the principle is dormant-by-design and re-arms on re-introduction (e.g. assert-and-log «0 hard-fail scripts — dormant», so a reader sees green≠active). **Option B** → accept as documented-dormant (the docstring already states it). Maintainer decides whether a green-but-dormant principle is acceptable signal.

### §4.2 DN-T3-2 — `20-bundle-classification.paired-negative.test.ts` never runs the real checker (SUSPECT)

- **Evidence:** the file imports only `vitest` ([20a:16](../../../packages/core/principles/20-bundle-classification.paired-negative.test.ts)); `brokenBundleNoFilter/NoCap/NoOverlap` ([20a:19/:29/:39](../../../packages/core/principles/20-bundle-classification.paired-negative.test.ts)) build markdown strings locally and every `expect` (e.g. [20a:105](../../../packages/core/principles/20-bundle-classification.paired-negative.test.ts) `.toBe(7)`) inspects that self-built string. The real `bundle-curate.sh` is never invoked, and the positive file's checks are never imported.
- **What slips past:** the docstring claims it «proves the positive checks are non-tautological», but it asserts properties of fabricated rows — it would not catch a genuinely-broken `bundle-curate.sh`. The real non-tautology proof lives in the **positive** file `20-bundle-classification.test.ts` (REAL — runs the real binary against fixtures with divergent expected counts).
- **Option A** → re-point the paired-negative at the real helper (run `bundle-curate.sh` on a deliberately-broken copy / fixture, assert the positive checks fail). **Option B** → keep the positive file as the sole non-tautology proof and demote this file to a documented simulation (rename / re-docstring so it does not over-claim). Maintainer decides.
