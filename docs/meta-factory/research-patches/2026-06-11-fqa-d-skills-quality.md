## §findings

### F1 — MAJOR — principle 15 enumerates gitignored installer skills → local/dogfooding RED while CI is green
`enumerateSkills()` (`15-*.test.ts:110-124`) uses `readdirSync` over roots `['.claude/skills','skills']` — **filesystem presence, not git-tracked status**. In this worktree (an installer-populated clone) the test is RED:
```text
$ npx vitest run packages/core/principles/15-skill-paired-negative.test.ts
.claude/skills/aif-evolve/SKILL.md: missing "## Without this skill" section
… (25 violations) …
expected [ …(25) ] to have a length of +0 but got 25
 Test Files  1 failed (1)   Tests  1 failed | 6 passed (7)
```
All 25 violators are gitignored `aif-*` installer skills (`.gitignore:36-37`). A **clean CI checkout** enumerates only the 10 tracked skills (`git ls-files .claude/skills/` → 8 dirs, all with a block or exempt; `skills/` → 2, both exempt) → **green**. `make self-audit` → `principles-meta-tests` → `npm run test:principles` → `vitest run principles/` (`Makefile:3,11-12`; `packages/core/package.json:27`) includes principle 15, so **`make self-audit` fails locally in any environment that ran the AIF installer**.
**Impact:** directly dents invariant #2 ("recursive self-application green") for a dogfooding maintainer — 25 false violations unrelated to their change; the gate gives a different verdict in exactly the environment where the framework is actually used.
**Falsifies if:** a clean `git clone` (no installer run) + `npm run test:principles` shows red, OR the `aif-*` skills are tracked. Both checked false: `git check-ignore -v .claude/skills/aif-plan/SKILL.md` → `.gitignore:37`; `git ls-files` omits them.
