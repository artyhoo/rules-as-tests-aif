### Problem 1 — Scope predicate (C4)

**Adversarial test findings:**

Ran `git log --oneline -20 main` and `git log --oneline origin/main..HEAD`.

From last 20 commits on `main`, candidates touching discipline-file globs:

| Commit | Subject | Glob hit | `## §` FP risk |
|--------|---------|----------|----------------|
| `4b6ebf6` | `test(principles): length sentinel on parsed SHIPPED_DOCS` | `packages/core/principles/*.test.ts` | **None** — TypeScript files cannot contain literal `## §` lines as code; the inner predicate `^+## §` cannot fire on `.ts` diffs |
| `4710083` | `test(principles): add SHIPPED_DOCS ↔ REQUIRED_HEADER_DOCS drift detector` | `packages/core/principles/*.test.ts` | **None** — same TypeScript constraint above |

No commits in the 20-commit sample touched `.claude/rules/*.md` or `.claude/skills/*/SKILL.md`.

**Structural observation on C4 and TypeScript files:** The predicate `git show "$sha" | grep -qE '^\+## §'` requires a diff line literally starting with `+## §`. TypeScript code cannot contain such a line unless it's inside a string literal or comment — in practice impossible for test files like `packages/core/principles/*.test.ts`. This means the `packages/core/principles/` glob effectively contributes zero FP surface — it only triggers when a TypeScript test file adds `## §` as literal text, which has never happened and is unlikely. This is a **structural reinforcement** of the C4 low-FP estimate: the glob that covers the most frequently-edited discipline surface (principle tests) is effectively neutralized by the inner predicate.

For `.claude/rules/*.md` and `.claude/skills/*/SKILL.md`, the only realistic FP scenario is a section rename (adds `+## §New Name`, removes `-## §Old Name`) or a structural reorganization without new discipline content. No such commits appear in the 20-commit sample.

**Sample size note:** Reviewer examined 20 main commits, not the "last 50" referenced in the research. The 20-commit window covers the most recent work including the principle-test commits that are the highest-frequency FP candidates. The adversarial test is directionally complete (the key structural insight — TypeScript inner predicate immunity — does not depend on sample size), but the full 50-commit test remains a recommended pre-promotion verification for 7.6.c.

**Estimated FP rate from sample:** **LOW** — 0/20 = 0% FP observed; structural analysis suggests `packages/core/principles/` glob contributes zero FP by construction; `.claude/rules/*.md` FP requires section rename which is infrequent; estimated <2% over full 50-commit history.

**Verdict: GO** — C4 predicate is tractable. With D1 (warn-only 30 days) as the implementation default, any residual FP surface produces warnings, not blocks, allowing calibration before promoting to `exit 1`. This is F1 shape (all 4 problems tractable), not F2 — the warn-only is a calibration window, not a permanent downgrade.

### Problem 3 — Trailer format (T1)

**Pre-push hook verification (direct code reading):**

`.husky/pre-push` lines 155–203 (`pa_check_trailer()` function):

```bash
pa_check_trailer() {
  local sha="$1" body line payload rationale
  body=$(git show -s --format=%B "$sha")
  local found_any=0
  while IFS= read -r line; do
    case "$line" in
      Prior-art:*)
        found_any=1
        payload="${line#Prior-art:}"
        ...
      ;;
    esac
  done <<< "$body"
```

**Confirmed:** `pa_check_trailer()` uses `case "$line" in Prior-art:*)` — line-anchored bash case matching, NOT `git interpret-trailers`. The function reads the commit body line by line and pattern-matches on the `Prior-art:` prefix.

This directly validates T1: a new `s17_check_trailer()` function using `printf '%s\n' "$body" | LC_ALL=en_US.UTF-8 grep -qE '^§1\.7:'` is fully consistent with the existing implementation pattern. It does NOT require refactoring `pa_check_trailer()` to `git interpret-trailers` (T4 refactor path). The two parsers are independent — each scans the full commit body for its own key.

The hook currently has 8 numbered sections (ending with `# ── 8. lychee offline link check`). The §1.7 check fits cleanly as section 9.

**Verdict: GO** — T1 tractable. Parallel grep-based function is structurally consistent with existing hook. No refactor. F3 path (parser refactor) not needed.

## §4 — Write-time fixups (items 7.6.c must address)

1. **Run full 50-commit adversarial test at implementation time.** Reviewer covered 20 main commits. 7.6.c implementer should run `git log --oneline -50 main` and cross-reference against the three discipline-file globs. If any FP is found, document in commit message; if rate >5%, default to `S17_WARN_ONLY=true` permanently (F2 shape) rather than as a calibration window.

2. **D3 allow-list must be documented in hook comments.** The three allowed prefixes (`docs(research-patches):`, `chore(snapshot-regen):`, `chore(prior-art-update):`) must be listed in the hook's inline comment block (not just in this research doc). Maintenance surface: when a new safe prefix is discovered, the comment is the SSOT for updating the allow-list.

3. **UTF-8 locale guard required in implementation.** The `§` character (U+00A7, 0xC2 0xA7 in UTF-8) requires `LC_ALL=en_US.UTF-8 grep -qE '^§1\.7:'` — NOT bare `grep -qE '^§1\.7:'` — to guard against `LC_ALL=C` environments (e.g. some CI runners). This is confirmed by the research and must be in the implementation, not just noted.

4. **7.6.c Commit 2 (SKILL.md update) self-check.** If the SKILL.md update adds a new `## §` heading to describe the 4th active layer, the commit will be in push range AND touch `.claude/skills/self-reflection/SKILL.md` AND add `## §`. The C4 predicate fires on itself. B1 trailer (`§1.7 Bootstrap: updating ladder description from 3 active → 4 active layers — §1.7 check not yet fully calibrated at this commit`) resolves this. Implementer must include B1 trailer on this commit.

5. **Section numbering.** The hook currently ends at `# ── 8. lychee offline link check`. The new §1.7 check should be `# ── 9. §1.7 discipline trailer check` per existing convention. Do not renumber existing sections.
