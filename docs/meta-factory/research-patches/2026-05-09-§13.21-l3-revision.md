```typescript
// Positive: file under exemption glob with intentionally-invalid content does not fail principle 09
it('files under packages/*/research/fixtures/**/with-drift/** are exempt', () => {
  const fixturePath = 'packages/core/research/fixtures/drift/with-drift/skills/rules-as-tests/SKILL.md';
  expect(isExempt(fixturePath)).toBe(true);
  // file intentionally lacks Authoritative-for header (with-drift fixture intent)
});

// Mutation: without exemption, fixture file would fail principle 09 (anti-tautology)
it('mutation: without exemption, with-drift fixture lacks authority header', () => {
  const fixturePath = resolve(REPO_ROOT, 'packages/core/research/fixtures/drift/with-drift/skills/rules-as-tests/SKILL.md');
  if (!existsSync(fixturePath)) return; // skip if fixture relocated
  const content = readFile(fixturePath);
  expect(hasAuthorityHeader(content)).toBe(false);
  // confirms: rule actually catches missing headers; exemption is the only reason
  // fixture passes the suite
});
```

## Prevention

Operationalisable rule, applies to any L3-style plan that introduces a discipline extension across multiple file surfaces:

> Before closing the plan, run §1.7 forward 6 layers + backward 6 steps **against the plan itself**. Forward: every existing project discipline (R/principles/SSOT/triggers/doc-authority/user-value-goal) checked, paths verified to exist (`find`/`ls` evidence), canonical lists diffed for duplicates. Backward: complete sweep of all artefacts under the new rule's scope (not §1.5 floor of 3-5; the *complete* set per `find` over scope), exemption mechanism specified as code, exemption itself has positive+mutation meta-test pair.

The bootstrap exemplar [`2026-05-09-recommendation-skips-own-discipline.md`](2026-05-09-recommendation-skips-own-discipline.md) was sufficient to justify §1.7 existence; this revision is the second-occurrence exemplar showing §1.7 catches the gap when applied (see companion patch [`2026-05-09-§13.21-l3-self-review.md`](2026-05-09-§13.21-l3-self-review.md)).
