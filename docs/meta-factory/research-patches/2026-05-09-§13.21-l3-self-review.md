## §2.5 candidate observation: H7 «skill consumption blindness»

**Pattern:** project-internal skill (e.g. [.claude/skills/self-reflection/](../../../.claude/skills/self-reflection/SKILL.md)) carries auto-trigger description that matches the current drafting context — keyword overlap is exact, scope overlap is exact — yet the skill is **not engaged** during drafting. The skill ships and lives, but the draftsman's attention frame doesn't load skill descriptions; auto-trigger requires the harness or operator to surface the skill, and absent that lift, the skill is invisible to the drafting session.

**Distinction from `#recommendation-skips-own-discipline` (parent):** parent says «discipline isn't applied to its own act of formation». H7 candidate says «discipline *is* materialised as a skill, *is* loadable, *has* matching keywords — yet doesn't load». Mechanism difference: parent is meta-cognitive (no representation of self-as-object); H7 is consumption-side (representation exists, retrieval fails).

**Distinction from `#recursive-self-application-gap` (sibling):** sibling is bottom-up vs top-down at the discipline-layer. H7 is at the skill-layer — a finer-grained mechanism within the broader pattern.

**Status:** 1/3 occurrences. Logging here; if a 2nd + 3rd occurrence surface in subsequent sessions, distill into [`phase-research-coverage.md §4` anti-pattern catalog](../../../.claude/rules/phase-research-coverage.md) per the standard 3-patch threshold (mirrors AIF `/aif-evolve`).

**Mitigation if promoted:** before drafting any plan/recommendation that introduces or extends discipline, run `ls .claude/skills/` + read first 5 lines of each `SKILL.md` `description:` field; if any description's auto-trigger keywords overlap with the current task's framing, **explicitly engage the skill** — do not rely on harness auto-trigger.

## Prevention

PRIORITY CHECK rule, applied before closing any L3-style multi-surface discipline plan:

> Before closing the plan, walk §1.7 forward 6 layers + backward 6 steps **against the plan itself** with **explicit evidence** (`find`/`grep`/`ls` output) for each layer. If a layer's evidence is «memory of project state» — that layer fails. Re-check against current branch state.
>
> Additionally, before drafting: `ls .claude/skills/` + scan auto-trigger descriptions; if a skill's keywords overlap the task framing, engage the skill explicitly. Auto-trigger is a hint, not a hook.
