## §4 — Cross-mechanism composition coherence

Question: do B1 + B2 + B3 stack into ONE cohesive flow, or do they conflict?

**Flow trace (verbatim from kickoff §1 footer):** B1 produces candidate list → B2 surfaces for approval → B3 executes approved subset autonomously.

**Coherence check (per recommendation per sub-mechanism above):**

| Stage | Adopted primitive | Output handoff to next stage |
|---|---|---|
| B1 → B2 | Renovate `packageRules`-shaped filter (vocab) + own bash eligibility filter (BUILD) | Emits structured candidate list `[{idx, item-path, classification, est-cost, diff-scope, file-overlap}]` |
| B2 → B3 | Argo `suspend` + `enum` parameter (ADAPT) + bulk-actions «milestone pack + toast-revert» (vocab) + Renovate dashboard single-interface (ADAPT) | Consumes candidate list, emits approved subset `[idx]` or STOP |
| B3 execution | SDD sequential dispatch (ADOPT) + Argo `failFast: true` + `retryStrategy.limit` (vocab) + `just &&` (vocab) | Consumes approved subset; emits per-item PR + halt-on-first-red incident |

**Conflict scan:**
- **No conflict** between B1's filter-output shape and B2's input expectations — both speak «list of items with attributes».
- **No conflict** between B2's approval-output and B3's input expectations — both speak «subset of idx».
- **Implicit dependency** B3 → infrastructure: B3 polls `gh pr checks` + auto-merge — requires PR auto-merge already enabled on project (per [`project_automerge_staging_plan`] memory — staging is trunk since 2026-05-22, auto-merge on `ci-success`). **Pre-condition satisfied; no new infra needed.**
- **One coherence risk: file-overlap conflict detection.** B1 must surface file-overlap between bundled items (e.g. item 2 and item 4 both edit `.claude/skills/X/SKILL.md`); B3's sequential dispatch with auto-merge means item 4's branch may be BEHIND-staging after item 2 merges. Either: (a) B1 rejects file-overlap bundles entirely (simple), or (b) B3 rebases per item before dispatch (complex — undoes parallel-subwave-isolation discipline). **Recommendation: B1 rejects file-overlap bundles (option a) — keeps B3 stupid-simple.** Documents in §5.

**Verdict §4: B1+B2+B3 compose coherently** given (i) B1 rejects file-overlap bundles, (ii) infrastructure (auto-merge, staging-trunk) pre-existing. No structural conflict surfaced.
