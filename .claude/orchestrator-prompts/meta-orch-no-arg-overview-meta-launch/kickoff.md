```markdown
- <repeat for each touched artefact>
```

**Hard rules (each historically failed the gate):**

1. **Heading depth = H3 (`###`), NOT H2 (`##`).** Awk regex matches `^### §1\.7 ...`. H2 silently fails. (6× recurrence as of 2026-05-25: PR #58/#69/#105/#111/#212/#215.)
2. **Word «applied» is required.** Headings without «applied» fail the regex.
3. **≥40 non-whitespace chars in EACH section body** (awk-counted, post-strip).
4. **≥1 file:line citation per section** matching regex `[^[:space:]]+\.[a-z]+:[0-9]+`.
5. **Mechanical-maintenance escape hatch:** `### §1.7 Skipped: <rationale ≥60 chars>`. Rationale ≥60 chars and specifies *why* this PR has no discipline surface.

**Pre-flight grep — Worker MUST run BEFORE `gh pr create`:**

```bash
echo "$PR_BODY" | grep -cE '^### §1\.7 (Forward|Backward)-check applied'  # must be 2 (or 1 for "Skipped:")
echo "$PR_BODY" | awk '/^### §1\.7 Forward-check applied/{c=1;next} /^###/{c=0} c' | tr -d '[:space:]' | wc -c  # must be ≥40
echo "$PR_BODY" | awk '/^### §1\.7 Backward-check applied/{c=1;next} /^###/{c=0} c' | tr -d '[:space:]' | wc -c  # must be ≥40
echo "$PR_BODY" | grep -cE '[^[:space:]]+\.[a-z]+:[0-9]+'  # must be ≥2
```

If any check fails → fix the body before pushing. CI catches the same failure but burns wall-clock; pre-flight is free.
