**Hard rules** (each one has historically failed the gate):

1. Heading depth = **H3** (`###`), NOT H2. Memory: 6× recurrence (#58/#69/#105/#111/#212/#215).
2. Word «applied» is required.
3. ≥40 non-whitespace chars in each section body.
4. ≥1 file:line citation per section.
5. Mechanical-maintenance escape hatch: `### §1.7 Skipped: <rationale ≥60 chars>`.

**Pre-flight grep — Worker MUST run BEFORE `gh pr create` (only if §4b triggered):**

```bash
echo "$PR_BODY" | grep -cE '^### §1\.7 (Forward|Backward)-check applied'  # must be 2
echo "$PR_BODY" | awk '/^### §1\.7 Forward-check applied/{c=1;next} /^###/{c=0} c' | tr -d '[:space:]' | wc -c  # must be ≥40
echo "$PR_BODY" | awk '/^### §1\.7 Backward-check applied/{c=1;next} /^###/{c=0} c' | tr -d '[:space:]' | wc -c  # must be ≥40

```

**If verdict is DEFER / REJECT / patch does NOT touch `prior-art-evaluations.md`:** §4b does NOT trigger — the PR body should still note in §closure: «§4b path-check: patch touches only `docs/meta-factory/research-patches/**` (not in §4b path list) → §4b does not apply.»

Reviewer discipline (`reviewer-discipline.md §2`):

1. Reads the diff + umbrella kickoff §1–§6 acceptance criteria.
2. Verifies every Q1–Q8 has command/file:line/DeepWiki excerpt evidence (counters T3 + T-PM-RP-A).
3. Verifies §3 prior-art extension was actually run (4 items: AIF /aif-evolve, state.md measurement, Tier-1 sweep, 6-item coverage) — counters T-PM-RP-B.
4. Verifies verdict is one of `ADOPT-A` / `ADOPT-B` / concretely-specified hybrid / `DEFER-with-precondition` / `REJECT-with-structural-reason` — NOT a punt («further research needed»).
5. Verifies T15 self-application — Track-row admission path declared in the patch.
6. For any strategy fork surfaced by the R-phase findings (e.g. «Direction A and B both satisfy constraints — maintainer prefers?»), emits DECISION-NEEDED with Option A / Option B consequences — does NOT pick.
7. Emits GO / REVISE / STOP verdict with BLOCKER / MAJOR / MINOR classification.

**Verdict routing:**

- **GO** → maintainer merges the PR.
- **REVISE** → Worker fixes findings, Phase -1 re-runs. After 3 REVISE cycles → escalate to maintainer (§11 F5).
- **STOP** → escalate to maintainer; halt before merge.
