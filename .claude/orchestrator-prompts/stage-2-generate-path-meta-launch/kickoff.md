## §4 Sub-wave dispatch instructions — S2 (L3-live menu-picker)

**Deliverable:** a `live` synthesis path under `packages/core/synthesizer/` where an LLM **picks rules from the menu** of the frozen curated Next `ResearchPlan`, emitting a `SynthesisPlan` — exercising the existing L4 on non-curated provenance for the first time.

**Build (Path A only — LLM configures existing plugins, never authors TypeScript):**

1. **Menu-picker prompt + plan contract** — given the frozen Next `ResearchPlan` (`store/next/`), the LLM selects which rules/configs to include and emits a `SynthesisPlan` conforming to the existing schema (so `validate()` consumes it unchanged). The menu = the curated recipe's rule set; the LLM's job is "which of these, in which config", not invention.
2. **Proving-case test (the testable surface — per spec §8 / T-S2-B, assert the L4 verdict, NOT the LLM's exact bytes):**
   - **(a) L4 accepts:** `validate(plan).ok === true` (`validator/validate.ts:18`) on the LLM-sourced `SynthesisPlan`.
   - **(b) Regression-match:** the validated plan's rule set reproduces the curated Next recipe (`recipes/nextjs-app-router.json`) — empty diff, or an **explainably** non-empty diff documented in the test.
3. **Paired valid/invalid** (principle 02) + AST-over-grep where applicable (principle 03). The generated rules are held to the same gate the framework enforces on consumers (T15).

**Hard constraints (non-negotiable):**

- **L4 + L5 byte-identical** — no edits to `packages/core/validator/` or `installer/`. Stage 1 already hardened the gates; Stage 2 only feeds them new-provenance input. Touching `validate.ts` = `#parallel-evolution-creep` (T-S2-A).
- **L2 frozen** — use the curated `store/next/` plan as-is; no live `web_search` (that is Stage 3).
- **No paid LLM in CI** (`no-paid-llm-in-ci.md`) — the menu-pick runs **session-bound / install-time on the operator's own subscription**; the committed test must NOT make an API-billed call in CI. The deterministic surface CI runs is the **L4 verdict on a captured/fixture plan**, not a live LLM round-trip.
- **Capability-commit discipline** — new code under `packages/core/` carries a `Prior-art:` trailer citing the Stage-0 SSOT entries (#154–#168; `/aif-evolve` = REFERENCE → L3-live stays BUILD) per `CLAUDE.md`.

**Acceptance (this sub-wave):** for the Next proving-case, an LLM-sourced `SynthesisPlan` passes the existing L4 **and** reproduces the curated recipe (empty/explainable diff), demonstrated by **command output** (not prose). `make self-audit` stays green (curated Next path byte-identical).
