**Hard rules (each has historically failed the CI gate):**

1. Heading depth = H3 (`###`), NOT H2.
2. Word «applied» is required in both headings.
3. ≥40 non-whitespace chars in EACH section body.
4. ≥1 `file:line` citation per section (regex `[^[:space:]]+\.[a-z]+:[0-9]+`).
5. Mechanical-maintenance escape hatch: `### §1.7 Skipped: <rationale ≥60 chars>` only for purely structural PRs.

Pre-flight grep before `gh pr create`:

```bash
echo "$PR_BODY" | grep -cE '^### §1\.7 (Forward|Backward)-check applied'  # must be 2
echo "$PR_BODY" | grep -cE '[^[:space:]]+\.[a-z]+:[0-9]+'  # must be ≥2
```

## §5 AI-traps active (per `ai-laziness-traps.md §3`)

See `.claude/rules/ai-laziness-traps.md §2` for full catalogue.

**Active canonical traps for dispatcher-skill:**

- **T11** (designing custom solution without prior-art check) — Stage 0 MUST run the BFR §3 mechanism before proposing the loop as BUILD; an upstream dispatch→monitor→harvest→advance loop may already exist.
- **T12** (skipping literature sweep because «I already know») — the agent-control-loop space (aif autoMode, Devin, Superpowers) updates fast; WebSearch at proposal time, not from training memory.
- **T16** (pattern-matching-on-name) — «monitor loop» / «dispatcher» upstream ≠ our problem class unless the X-vs-Y problem-class match is written explicitly.
- **T19** (handoff without own cold-QA) — run an adversarial cold-review of each diff before handoff; CI ≠ design review.
- **T20** (inline-verdict-without-evidence) — no ADOPT/BUILD/REJECT on the loop or on DN-A/DN-B without a preceding evidence-bearing tool call in the same turn.
- **T15** (self-application — MANDATORY) — `/dispatcher` runs the loop; Stage 2 eval IS the self-application gate (the skill must drive a real umbrella to done). The R-phase must ask «what would auditing this loop design look like?».

**Domain-specific trap (dispatcher-skill-specific, NOT in canonical catalogue):**

- **T-disp-A — «circular dogfood»:** tempted to dispatch Stage 0/1 *via aif* because aif is up. But this umbrella BUILDS the aif-control loop — using aif to build it means a half-working loop is in the dispatch path, contaminating the result. Counter: Stage 0/1 are CC-paste-tab only (umbrella §0 runner constraint); aif enters only at Stage 2 eval, as the *subject* not the *runner*.
