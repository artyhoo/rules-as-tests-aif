## §5 AI-traps active (per `ai-laziness-traps.md §3`)

See `.claude/rules/ai-laziness-traps.md §2` for full catalogue.

**Active canonical traps for mutation-discipline-umbrella:**

- **T1 — sampling shallow:** Stage 1 must run Stryker on FULL TS scope, not «sampled 3 files, looks fine». Force depth ≥ all files in current Stryker config.
- **T2 — methodology theatre:** Stage 1 finding doc must contain ACTUAL Stryker output excerpts + file:line cites — not «would detect» or «coverage looks good».
- **T3 — no prose-only findings:** every claim = command + output OR file:line citation. «Coverage is high» forbidden; «file X = 87% kill (12/14 mutants killed, 2 surviving on lines Y, Z)» required.
- **T9 — sampling EASY:** Stage 1 bash hook sampling must span recent + ancient hooks, not just M.4-fresh ones.
- **T10 — completeness ≠ what you looked at:** Stage 1 must enumerate full population (`ls .claude/hooks/*.sh | wc -l` = 9; `find packages/core -name '*.ts' | wc -l` for TS) before any coverage claim.
- **T11 — prior art:** Stage 2 B.1 mandatory — DeepWiki + WebSearch ≥3 phrasings for «bash mutation testing». Per BFR-default: REFERENCE > BUILD.
- **T14 — clean ≠ no theatre:** 100% kill rate on a file may mean «code is mutation-resistant by structure (trivial constants)» — distinguish from «test caught everything».
- **T15 — self-application:** the bash mutator itself must have its own .test.ts; recursive bootstrap explicitly documented.
- **T16 — pattern-matching-on-name:** «we have Stryker» ≠ «we have mutation coverage». Verify file-by-file what Stryker actually mutates (audit kickoff §0 already cites the explicit excludes).
- **T17 — preserve before delete:** D.6 (duplicate `deps-hash-check.sh`) — investigate before delete; one copy may have unique install-path purpose.
- **T19 — own QA before handoff:** every PR in this umbrella runs the mutator on its own changes before push; CI green ≠ mutation-tested.

**Domain-specific (`T-MUT-*`, NOT in canonical catalogue):**

- **T-MUT-A — «manual mutation-sanity ≡ automated mutation testing».** Tempted: claim «M.4 already mutation-tested because sub-agents did manual sanity». Counter: manual sanity = single-pass at write time. Automated = continuous in CI. They are NOT interchangeable — this umbrella exists because of the gap.
- **T-MUT-B — «high mutation score = bug-free test».** Tempted: assume that if a test kills 95% of mutants, it's good. Counter: mutation testing has known false-positive rate (equivalent mutants); a survived mutant may still be valid code. Audit findings MUST categorize survived mutants by `legitimate-equivalent` vs `real-gap`.

> **Anti-pattern warning:** blanket «see ai-laziness-traps.md» without the enumeration above = T7 violation (pattern-matching the prompt instead of reasoning against it). This meta-kickoff enumerates 11 canonical T-numbers + 2 domain-specific T-MUT-* traps; principle 12 validates the §5 syntax.
