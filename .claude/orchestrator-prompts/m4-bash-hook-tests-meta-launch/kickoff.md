## §3 Stage gates (real git checks — NOT in-memory FIFO)

### Stage 1 → DONE (no Stage 2 in M.4)

M.4 has only Stage 1 — 6 parallel sub-waves with no internal dependency edges. Stage 1 = DONE when ALL 6 PRs merged:

```bash
gh pr list --search "is:merged head:test/m4- base:staging created:>=2026-05-24" \
  --json number,title,mergedAt,headRefName --limit 10
```

Expected: 6 merged PRs (one per `test/m4-<hook>` branch).
If <6 → not done; wait or investigate which sub-wave stalled.

### Downstream stage gate (M.5a unblock)

M.4 completion unblocks Track M.5a (top-3 edit-time hooks: manifest render-drift / research-patch §1.7-substance / actionlint per-yml). M.5a dispatch waits on:

```bash
gh pr list --search "is:merged head:test/m4- base:staging created:>=2026-05-24" \
  --json number --limit 10 | jq 'length == 6'
```

Returns `true` → M.5a can dispatch.

**Phase -1 cold-review before declaring M.4 done is MANDATORY** — see §7 below.

## §5 AI-traps active (per [ai-laziness-traps.md §3](../../../.claude/rules/ai-laziness-traps.md))

See `.claude/rules/ai-laziness-traps.md §2` for full catalogue.

**Active canonical traps for m4-bash-hook-tests: T3, T5, T11, T13, T15, T19.**

- **T3** (no prose-only findings): each test must cite the hook source with file:line and the behaviour the assertion captures.
- **T5** (no implementation bundled into research / scope-expansion): hook bug found mid-test → surface in PR body, don't fix in M.4 PR.
- **T11** (prior-art before building): REFERENCE the `check-hook-marker.test.ts` precedent; cite in new test docstring; don't reinvent fixture-spawn shape.
- **T13** (ADOPTED items aren't zero-work): vitest is adopted, but per-hook fixture is still our work; assert exit code AND payload shape, not just exit.
- **T15** (self-application): the new test files are leaves; tests do NOT need their own tests (infinite regress). Note in PR body to pre-empt T15 confusion.
- **T19** (own cold-QA before handoff): each sub-wave runs `pnpm test` + manual mutation-sanity step (umbrella §5 step 7) BEFORE pushing — CI green ≠ design verified.

**Domain-specific traps (m4-bash-hook-tests-specific, NOT in canonical catalogue):**

- **T-M4-A — «bash hook = test the bash directly»** (T16-shape).
  Tempted: write `.bats` tests or `bash hook.sh < fixture` shell-only invocations. Counter: 3-hook precedent uses **TS + `spawnSync`** (`check-hook-marker.test.ts:25-30`); adding `bats` splits the test harness and forces CI dep install. ADOPT existing pattern.
- **T-M4-B — «just test exit code 0/1»**.
  Tempted: minimal assert on exit code only. Counter: hooks have BOTH exit code AND `stdout`/`stderr` semantics — Stop hook contract = `reason` → MODEL + `systemMessage` → USER (per [memory `project_eot_hook_redesign_approved`](~/.claude/projects/-Users-art-code-rules-as-tests-aif/memory/project_eot_hook_redesign_approved.md)); PostToolUse = JSON `additionalContext`. Exit-only assertions miss the channel-injection regressions EOT contract had TWICE (false-confirm). Assert exit AND payload shape.
- **T-M4-C — «6 hooks → 6 identical tests in 6 hours»** (T16-shape).
  Tempted: copy-paste skeleton 6×. Counter: each hook has different trigger contract (PostToolUse vs UserPromptSubmit vs Stop vs PreToolUse). Read each hook source (umbrella §5 step 3) before writing its test.

> **Anti-pattern warning:** blanket «see ai-laziness-traps.md» without enumeration above = T7 violation. The above enumeration + 3 domain-specific traps satisfies principle 12 ([12-ai-laziness-traps.test.ts](../../../packages/core/principles/12-ai-laziness-traps.test.ts)).
