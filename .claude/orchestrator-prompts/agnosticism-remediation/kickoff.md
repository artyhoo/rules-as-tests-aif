## §1 Headline result — the framework is ALREADY largely agnostic

**This is the load-bearing outcome and it is the opposite of the going-in assumption.** The initial prose recon claimed orchestration (`pipeline`/`dispatcher`) would HARD-BREAK off Claude Code. **Test-backed verification refuted that.** What is *proven* portable (not asserted — run/grepped):

| Proven PORTABLE (test-backed) | Evidence |
|---|---|
| 6 shipped agents (`agents/*.md`) | each names "Claude Code, Cursor, Codex, Aider, …" + uses only `read_file`/`gh`/`git`; grep for CC-only primitives = 0 |
| 4 orchestration skills' substrate (`pipeline`/`dispatcher`/`aif-doctor`/`template-audit`) | helpers self-resolve via `$(dirname "${BASH_SOURCE[0]}")` (`priority-score.sh:82`); `CLAUDE_SKILL_DIR` in helpers = comments-only (`parse-override-flags.sh:33`, `integer-name-guard.sh:19`); `dispatcher/SKILL.md:203` ships a `CC-absent degradation` section |
| substrate (eslint/vitest/dep-cruiser/stryker/tsconfig configs) | zero `CLAUDE_*`/`ANTHROPIC` coupling; already runs green in GitHub Actions = a CC-absent env |
| git hooks (`.husky/pre-commit`, `pre-push`) | git-driven (`core.hooksPath`), no CC coupling; portable `pre-push.fallback.sh` present |
| paid surface | transport opt-in surfaced (`setup-runtime-bridge.sh:77`); no real `ANTHROPIC_API_KEY` usage in `.github/workflows/` (only a comment asserting absence) |

**Consequence: the remediation umbrella is SMALL and targeted, not giant.** Three modest finding-classes remain (§2). The expensive "rewrite orchestration to be portable" work the going-in framing implied is **not needed** — it is already portable-with-graceful-degradation.
